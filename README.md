## Propósito del Proyecto

El objetivo principal de este espacio es aplicar y estandarizar los componentes esenciales que todo proyecto de código abierto o empresarial debe contemplar para asegurar una colaboración organizada:

* **Flujo de Trabajo Estructurado (Workflow):** Practicar la gestión y uso de ramas (*branches*), el análisis de conflictos y el ciclo de vida completo de los *Pull Requests*, desde su apertura y revisión de código hasta su integración final.
* **Lineamientos de Contribución:** Establecer criterios técnicos y organizacionales claros mediante el archivo de [Contribución](CONTRIBUTING.md) para homogeneizar la calidad del código entregado.
* **Gobernanza del Proyecto:** Incorporar un [Código de Conducta](CODE_OF_CONDUCT.md) que garantice un entorno de colaboración profesional y respetuoso.
* **Marco Legal:** Definir los términos de uso, modificación y distribución del software a través de una licencia de código abierto regulada.

---

## Estándares de Colaboración

Para el desarrollo de las actividades en este repositorio, se consideran indispensables los siguientes pilares de ingeniería de software:

1. **Consistencia de Código:** Mantener la legibilidad y estructura del código de acuerdo con los estándares de la comunidad (tales como PEP 8 para Python).
2. **Documentación:** Asegurar que cualquier modificación o nueva funcionalidad se refleje adecuadamente en la documentación técnica del proyecto.
3. **Validación:** Comprobar el correcto funcionamiento de los módulos de cálculo antes de solicitar la integración de cambios a la rama principal.
4. **Aislamiento de Cambios:** Restringir el desarrollo directo sobre la rama `main`, promoviendo el uso estricto de ramas de características de forma aislada.
