# Contribution Guidelines

Todas las contribuciones, informes de errores, correcciones de errores, mejoras en la documentación, mejoras y ideas son bienvenidas.

## Estilo de código
Sigue el estilo y las convenciones utilizados en la base de código existente. Esto ayuda a mantener la coherencia en el proyecto.

## Documentación
Asegúrate de que tus contribuciones estén bien documentadas. Incluye comentarios donde sea necesario y proporciona una descripción clara de tus cambios en el pull request.

## Pruebas
Antes de enviar un pull request, asegúrate de que tus cambios hayan sido probados exhaustivamente. Incluye casos de prueba relevantes y revisa que los tests existentes pasen.

## Seguimiento de issues
Consulta el issue tracker para ver problemas abiertos o solicitudes de funcionalidades. Si vas a trabajar en algo, comenta para informar a otros.

## Revisión de código
Todas las contribuciones pasan por un proceso de revisión de código. Sé abierto a los comentarios y dispuesto a hacer cambios si es necesario. Las revisiones ayudan a mantener la calidad y coherencia del código.

¡Gracias por tu contribución!
